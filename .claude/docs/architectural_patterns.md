# Architectural Patterns

Recurring design decisions and conventions found across the RoboRecruit codebase.

## 1. Views vs. Components split

`src/views/` holds page-level components — one per route, responsible for layout and composing sub-components. `src/components/` holds reusable UI pieces. Views own the data wiring; components are display-only or close to it.

Examples:
- `src/views/LandscapeView.tsx` — composes `FilterPanel` + `CompanyGrid` + `CompanyModal`
- `src/views/JobsView.tsx` — composes `JobTable`, manages local search/level filter state

**Convention:** never import one view from another. Each view is self-contained.

## 2. Filtering via custom hooks + Zustand

Global filter state (domain, technology, stage, hasOpenJobs) lives in `src/store/filterStore.ts` (Zustand). Derived data is computed in custom hooks, not in components or the store.

Pattern:
```
Zustand store (raw filter selections)
  → custom hook (reads store + static data → derived array)
  → view/component (consumes array, no filtering logic)
```

Files:
- `src/store/filterStore.ts` — toggle actions, `clearAll`
- `src/hooks/useFilteredCompanies.ts` — multi-predicate filter over `companies`
- `src/hooks/useFilteredJobs.ts` — keyword + level filter over `jobs`
- `src/hooks/useCompanyById.ts` — single-item lookup

**Convention:** all filtering logic belongs in a hook; components call the hook and render the result.

## 3. Static data + auto-generated data, never mixed

`src/data/companies.ts` is hand-authored and imported directly. `src/data/jobs.ts` is machine-generated by `scripts/fetch_jobs.py` and committed by CI — it has a `// AUTO-GENERATED` header comment. Both are re-exported via `src/data/index.ts` along with derived lookup maps (`companiesById`, `jobsByCompany`).

**Convention:** never hand-edit `jobs.ts`. Derived indexes live in `src/data/index.ts`, not in individual components.

## 4. TypeScript enums as the single source of truth for domain vocabulary

`src/types/index.ts` defines all domain enums (`ProblemDomain`, `CompanyStage`, `JobLevel`, `TechCategory`). These drive both the data (`Company.problems`, `Job.level`) and the filter UI (`MultiSelect` option lists). No string literals are used where an enum value exists.

**Convention:** add new domains/tech-categories to `src/types/index.ts` first; update data and UI after.

## 5. Badge variants as a type-safe display contract

`src/components/ui/Badge.tsx` accepts a `variant` prop (`domain | tech | stage | level | count | remote`). Each slice of the domain has exactly one badge variant. Components never style category labels inline — they always delegate to `<Badge variant="...">`.

## 6. HashRouter for GitHub Pages compatibility

`src/App.tsx` wraps routes in `<HashRouter>` (not `<BrowserRouter>`). This is load-bearing for GitHub Pages, which cannot serve arbitrary paths. All internal links use React Router's `<Link>` and `useNavigate` — never `window.location`.

`vite.config.ts` sets `base: '/RoboRecruit/'` to match the GitHub Pages repo sub-path. Both must stay in sync when the repo is renamed.

## 7. Portal-based Modal

`src/components/ui/Modal.tsx` renders into a `document.body` portal via `ReactDOM.createPortal`. It handles `Escape` key and click-outside dismissal internally. Callers only manage an `isOpen` boolean + `onClose` callback.

**Convention:** all overlay/dialog UI should use this component rather than ad-hoc `position: fixed` divs.

## 8. SVG-based heatmap (no chart library)

`src/components/charts/TechHeatmap.tsx` builds the matrix with raw SVG `<rect>` and `<text>` elements. It does not use D3 for rendering — only for potential data transforms. Recharts is used elsewhere but was not chosen here because the custom cell-coloring and label rotation require direct SVG control.

## 9. Responsive layout via AppShell + Tailwind breakpoints

`src/components/layout/AppShell.tsx` controls the top-bar + sidebar + main-content triptych. Mobile sidebar is an overlay (fixed, `z-50`, blur backdrop) toggled by a boolean in `TopBar`. Desktop sidebar is always visible.

Grid breakpoints used consistently across the app: `sm:grid-cols-2`, `xl:grid-cols-3`, `2xl:grid-cols-4`.

## 10. CI data pipeline pattern

`update-jobs.yml` runs daily, calls the Python script, and auto-commits only when `src/data/jobs.ts` has changed (checked via `git diff --quiet`). The commit triggers `deploy.yml`, which rebuilds and deploys. This creates an implicit dependency: the deploy workflow must not interfere with the auto-commit author (`github-actions[bot]`).
